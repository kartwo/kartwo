## 2. 金额与数值
- 金额一律 **整数（分）**，类型 `int64`，绝不用浮点表示钱。
- 展示层再按币种换算/格式化。

## 3. 标识符
- 外部可见 ID 用 **Hashid（最小长度 8）或 UUID v7**。
- **不暴露自增主键**、不用雪花。内部主键可自增，但对外一律映射。

## 4. 数据库与迁移
- **纯 SQL 迁移文件，禁止 ORM AutoMigrate**。
- 迁移按版本号顺序命名、**幂等可重入**、向后兼容。
- 统一字段 `created_at / updated_at / deleted_at`（软删）。
- 所有查询**参数化**（防 SQLi），禁止字符串拼接 SQL。
- 数据层抽象需同时支持 SQLite 与 PostgreSQL。

## 5. 凭证与密钥
- 支付/SMTP/会话密钥**加密存后台**，**绝不硬编码、不进日志、不进 git**。
- 加密密钥来源做成接口：自部署=主口令派生；SaaS=平台托管。
- 导出包内 `config.enc` 保持加密。

## 6. 安全编码（硬约束）
- 输出转义防 XSS；写操作 CSRF 保护。
- 所有接口鉴权 + **对象级权限校验**（防 IDOR）。
- 上传：magic bytes 校验类型/大小、隔离不可执行、正确 content-type。
- 导入/抓取远程资源：**SSRF 防护**（禁内网地址、URL 白名单、超时与体积限制）。
- 强制 HTTPS + 安全响应头（HSTS/CSP/X-Frame-Options/X-Content-Type-Options）。
- 密码 argon2id（或 bcrypt）+ 登录限流。
- CI 必须含 govulncheck + 密钥泄漏扫描，**无高危方可合主干**。

## 7. 测试
- 核心逻辑（状态机/金额/库存/支付回调/导入幂等）**必须有单元测试**。
- 关键路径有集成测试，纳入 CI。

## 8. 资源许可
- 字体仅 **OFL / Apache-2.0**（Inter、Noto）；图标仅 **MIT/Apache/ISC/CC0**（Lucide、Tabler、Heroicons）。
- 禁商用/付费/许可不清/来源不明；每引入一个登记到 `LICENSES.md`。

## 9. 双模式纪律（架构级）
- 配额/计费/密钥来源/实例身份等分叉点，**只做接口 + 默认实现（自部署语义）**。
- 内核不写 SaaS 专属逻辑、不写控制面。控制面是 v2+ 的独立外壳。

## 10. 范围与改动纪律
- 一次只做当前里程碑范围，**不镀金、不顺手改无关代码**。
- 超出范围的决策回报 Derek，不自行假设。

## 11. Git 与提交
- 每里程碑一分支 `feat/m{n}-xxx`；验收过合主干并打 tag `v0.{n}.0`。
- 小颗粒提交，单提交单一意图，message 规范（如 `feat(product): 增加变体模型`）。
- 每轮收尾更新 `DECISIONS.md`（新决策）与 `PROGRESS.md`（进度），并在回报中列出。
